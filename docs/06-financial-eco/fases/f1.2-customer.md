# F1.2 — Customer (Cliente)

> **Fase:** F1.2 · **Esfuerzo:** Bajo · **Sesión:** 2/N
> **Depende de:** F1.1 ✅
> **Estado:** ✅ Completado

---

## 0. Objetivo

Crear el dominio **Customer (Cliente)** asociado a Company (`@BelongsTo`),
expuesto como `/odata/finance/customer-odata`. El cliente SAP (KUNNR-like)
pertenece a una sociedad (`companyId`) y es el primer dominio financiero con
relación de navegación resoluble vía `$expand`.

---

## 1. Modelo de datos

| Campo | Tipo | PK | FK | Descripción |
|-------|------|----|----|-------------|
| `id` | `STRING` | ✅ | | Código de cliente (p.ej. `"100001"`, `"100002"`) |
| `nombre` | `STRING` | | | Nombre / razón social |
| `companyId` | `STRING` | | → `companies.id` | Sociedad a la que pertenece |
| `pais` | `STRING` | | | País (p.ej. `"US"`, `"DE"`) |
| `createdAt` | `DATE` | | | Timestamp (Sequelize) |
| `updatedAt` | `DATE` | | | Timestamp (Sequelize) |

- Relación `@BelongsTo(() => CompanyOData)` navegable como `company`.
- Se añadió `@HasMany(() => CustomerOData)` en `CompanyOData` para navegación inversa (`customers`).
- Tabla física: `customers`.
