# F1.2 — Dominio `customer` (Cliente)

> **Sub-fase:** F1.2 · **Esfuerzo:** Bajo · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F1.1.

## 0. Objetivo
Crear `core/customer/` — cliente (KUNNR-like) con relación 1—N a `invoice`.

## 1. Modelo `core/customer/model/customer.odata.model.ts`
```typescript
@Table({ tableName: "customers" })
export class CustomerOData extends Model<CustomerOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) customerId!: string; // "C00001"
    @Column({ dataType: DataTypes.STRING }) name!: string;                           // "Acme Iberia S.A."
    @Column({ dataType: DataTypes.STRING }) vatId!: string;                          // "ESB12345678"
    @Column({ dataType: DataTypes.STRING }) city!: string;                           // "Madrid"
    @Column({ dataType: DataTypes.STRING }) paymentTerms!: string;                  // "30D"
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;

    @HasMany(() => InvoiceOData, { relation: [{ foreignKey: "customerId", sourceKey: "customerId" }] })
    invoices!: InvoiceOData[];
}
```
Nota: la navegación `@HasMany` a `InvoiceOData` se completa en F1.5; hasta entonces se omite o
se añade al cerrar F1.5 (evitar import circular: `invoice` importa `customer` y viceversa — usar
`() => InvoiceOData` lazy).

## 2. Resto del módulo
- `interface/customer.interface.ts` → `ICustomer`.
- `dto/customer.dto.ts` → `CustomerCreateDTO` / `CustomerUpdateDTO`.
- `controller/customer.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/customer.service.ts` → singleton.
- `main.ts` → barrel.

## 3. Registro
- `datasource.ts`: añadir `CustomerOData`.
- `odata.service.ts`: añadir `new CustomerODataController()`.
