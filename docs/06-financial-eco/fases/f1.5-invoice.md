# F1.5 — Dominio `invoice` (Factura cliente / SD)

> **Sub-fase:** F1.5 · **Esfuerzo:** Medio · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F1.1, F1.2, F1.7 (items) — se crea primero la tabla; la navegación a items se añade al cerrar F1.7.
> **Central:** es la entidad que más consultará el chatbot.

## 0. Objetivo
Crear `core/invoice/` — factura de venta (SD) con relaciones a `customer`, `company` y (en F1.7) `items`.

## 1. Modelo `core/invoice/model/invoice.odata.model.ts`
```typescript
@Table({ tableName: "invoices" })
export class InvoiceOData extends Model<InvoiceOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) invoiceId!: string;   // "I00001"
    @Column({ dataType: DataTypes.STRING }) docNumber!: string;                        // "1900000001"
    @Column({ dataType: DataTypes.STRING }) company!: string;                          // "1000"
    @Column({ dataType: DataTypes.STRING }) customerId!: string;                       // "C00001"
    @Column({ dataType: DataTypes.DATE }) date!: Date;
    @Column({ dataType: DataTypes.DATE }) dueDate!: Date;
    @Column({ dataType: DataTypes.DECIMAL }) netAmount!: number;
    @Column({ dataType: DataTypes.DECIMAL }) taxAmount!: number;
    @Column({ dataType: DataTypes.DECIMAL }) grossAmount!: number;
    @Column({ dataType: DataTypes.STRING }) status!: string;                           // PENDIENTE | PAGADA | VENCIDA
    @Column({ dataType: DataTypes.STRING }) currency!: string;                        // "EUR"
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;

    @BelongsTo(() => CustomerOData, { relation: [{ foreignKey: "customerId", targetKey: "customerId" }] })
    customer!: CustomerOData;

    @BelongsTo(() => CompanyOData, { relation: [{ foreignKey: "company", targetKey: "code" }] })
    companyRef!: CompanyOData;

    @HasMany(() => InvoiceItemOData, { relation: [{ foreignKey: "invoiceId", sourceKey: "invoiceId" }] })
    items!: InvoiceItemOData[];
}
```
Los `@BelongsTo`/`@HasMany` usan `() => ...` lazy para evitar imports circulares.

## 2. Resto del módulo
- `interface/invoice.interface.ts` → `IInvoice`.
- `dto/invoice.dto.ts` → `InvoiceCreateDTO` / `InvoiceUpdateDTO` (valida importes positivos, status enum).
- `controller/invoice.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/invoice.service.ts` → singleton; helper `computeStatus(dueDate)` para F3.
- `main.ts` → barrel.

## 3. Registro
- `datasource.ts`: añadir `InvoiceOData` (junto a `CustomerOData`, `CompanyOData`).
- `odata.service.ts`: añadir `new InvoiceODataController()`.
