# F1.8 — Dominio `payment` (Pago / transacción con clearing)

> **Sub-fase:** F1.8 · **Esfuerzo:** Medio · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F1.5.

## 0. Objetivo
Crear `core/payment/` — pago/transacción financiera que "clearea" facturas (PENDIENTE → PAGADA).
El `clearedInvoice` enlaza a `invoice` (o a `supplierinvoice` vía `partnerType`).

## 1. Modelo `core/payment/model/payment.odata.model.ts`
```typescript
@Table({ tableName: "payments" })
export class PaymentOData extends Model<PaymentOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) paymentId!: string;   // "P00001"
    @Column({ dataType: DataTypes.STRING }) company!: string;                         // "1000"
    @Column({ dataType: DataTypes.STRING }) partnerId!: string;                       // "C00001" o "S00001"
    @Column({ dataType: DataTypes.STRING }) partnerType!: string;                     // "C" (cliente) | "S" (proveedor)
    @Column({ dataType: DataTypes.DATE }) date!: Date;
    @Column({ dataType: DataTypes.DECIMAL }) amount!: number;
    @Column({ dataType: DataTypes.STRING }) method!: string;                         // "TRANSFER" | "CARD" | "CHECK"
    @Column({ dataType: DataTypes.STRING, allowNull: true }) clearedInvoice!: string; // "I00001" (FK lógico)
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;

    // Navegación opcional a invoice (solo cuando partnerType = "C")
    @BelongsTo(() => InvoiceOData, { relation: [{ foreignKey: "clearedInvoice", targetKey: "invoiceId" }] })
    invoice!: InvoiceOData;
}
```
Nota: `clearedInvoice` es FK lógica (no restricción SQL estricta) para soportar ambos socios.

## 2. Resto del módulo
- `interface/payment.interface.ts` → `IPayment`.
- `dto/payment.dto.ts` → `PaymentCreateDTO` / `PaymentUpdateDTO` (valida `partnerType` enum, amount > 0).
- `controller/payment.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/payment.service.ts` → singleton; helper `applyClearing` para F3.
- `main.ts` → barrel.

## 3. Registro
- `datasource.ts`: añadir `PaymentOData`.
- `odata.service.ts`: añadir `new PaymentODataController()`.
