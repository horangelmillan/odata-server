# F2.0 — Infraestructura de seed idempotente

> **Sub-fase:** F2.0 · **Esfuerzo:** Medio · **Depende de:** F1.
> **Actualiza:** `package.json`, crea `scripts/seed/`.

## 0. Objetivo
Crear el andamiaje de seed: loader JSON + orden de borrado/inserción determinista.

## 1. `scripts/seed/financial-seed.ts` (esqueleto)
```typescript
import { dataSource } from "../../src/common/service/odata/datasource.js";
// importar los 8 modelos
const ORDER_INSERT = [Company, Customer, Supplier, GlAccount, Invoice, InvoiceItem, SupplierInvoice, Payment];
const ORDER_DELETE = [...ORDER_INSERT].reverse();

async function clear() {
  for (const m of ORDER_DELETE) await m.destroy({ where: {}, truncate: true, cascade: true });
}
async function seed() {
  for (const m of ORDER_INSERT) {
    const rows = await loadJson(`data/${m.name}.json`); // IDs fijos
    await m.bulkCreate(rows);
  }
}
async function main() {
  const sequelize = dataSource.sequelizerAdaptor.sequelize;
  await sequelize.authenticate();
  if (process.argv.includes("--reset")) { await sequelize.sync({ alter: true }); }
  await clear(); await seed();
  console.log("financial seed done (idempotent)");
}
main();
```

## 2. `package.json` scripts
```json
"seed": "node --loader ts-node/esm scripts/seed/financial-seed.ts",
"db:reset": "node --loader ts-node/esm scripts/seed/financial-seed.ts --reset"
```

## 3. Criterios de aceptación
- [ ] `scripts/seed/financial-seed.ts` corre (aunque los JSON aún estén vacíos).
- [ ] `pnpm seed` / `pnpm db:reset` invocables.
