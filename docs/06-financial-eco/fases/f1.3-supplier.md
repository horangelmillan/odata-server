# F1.3 — Dominio `supplier` (Proveedor)

> **Sub-fase:** F1.3 · **Esfuerzo:** Bajo · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F1.1.

## 0. Objetivo
Crear `core/supplier/` — proveedor (LIFNR-like) con relación 1—N a `supplierinvoice`.

## 1. Modelo `core/supplier/model/supplier.odata.model.ts`
```typescript
@Table({ tableName: "suppliers" })
export class SupplierOData extends Model<SupplierOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) supplierId!: string; // "S00001"
    @Column({ dataType: DataTypes.STRING }) name!: string;                           // "Proveedor Global S.L."
    @Column({ dataType: DataTypes.STRING }) vatId!: string;                          // "ESA98765432"
    @Column({ dataType: DataTypes.STRING }) city!: string;                          // "Barcelona"
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;

    @HasMany(() => SupplierInvoiceOData, { relation: [{ foreignKey: "supplierId", sourceKey: "supplierId" }] })
    supplierInvoices!: SupplierInvoiceOData[];
}
```
(El `@HasMany` a `SupplierInvoiceOData` se completa en F1.6.)

## 2. Resto del módulo
- `interface/supplier.interface.ts` → `ISupplier`.
- `dto/supplier.dto.ts` → `SupplierCreateDTO` / `SupplierUpdateDTO`.
- `controller/supplier.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/supplier.service.ts` → singleton.
- `main.ts` → barrel.

## 3. Registro
- `datasource.ts`: añadir `SupplierOData`.
- `odata.service.ts`: añadir `new SupplierODataController()`.
