## 0. Objetivo

Aplicar el prefijo de namespace semántico `demo/` a los dominios de demostración
(`product`, `category`) mediante `getEndpoint() → /odata/demo/...`, según la
decisión **D8** del `00-plan-maestro.md`. Esto prepara el terreno simétrico para
los dominios `finance/` del ecosistema S/4HANA.

---

## 1. Ejecución intentada y hallazgo

Se modificó `src/core/product/controller/product.odata.controller.ts` y
`src/core/category/controller/category.odata.controller.ts` añadiendo
`endpoint: "demo/<model>-odata"` al `super({...})`, y se migraron las 67
referencias de ruta en 5 archivos de test de integración.

**Resultado:** `pnpm test` pasó de **143 passed** a **112 passed / 31 failed**.
La causa raíz **no eran los tests**, sino un acoplamiento de dominio en el
**shared kernel**: el código asume que la *ruta de exposición* (endpoint) es
idéntica al *nombre del modelo / EntitySet*.

### Puntos de acoplamiento (detalle en issue #3)

| # | Archivo:línea | Síntoma |
|---|---------------|---------|
| 1 | `src/common/service/odata/odata-write.routes.ts:22-25` | `endpointServices` keyed por `"product-odata"`; con endpoint `demo/product-odata` el `service` queda `undefined` → POST/PATCH/DELETE devuelven 404. |
| 2 | `src/common/middleware/batch.middleware.ts:184-218` (`resolveTarget`) | El `entitySet` se deriva del endpoint; con prefijo el `QueryParser` y los errores de `$batch` lo reciben roto. |
| 3 | `src/common/service/odata/odata-metadata.ts:89-109` | El EntitySet en `$metadata` se deriva de `et.$Endpoint`; con prefijo pasa a `demo/product-odata`, rompiendo el bootstrap del demo UI5 (que localiza el EntitySet por nombre de modelo). |

### Conclusión

El shared kernel funde *ruta de exposición* con *nombre de modelo* — un *leak* de
dominio que viola el principio de dominio agnóstico (AGENTS.md) y la definición de
shared kernel ("no contiene modelos ni controladores de dominio"). Mover carpetas
físicas es seguro; **cambiar el prefijo de ruta rompe el kernel**.

Se **revertieron** los cambios de F0.1 (rama `feature/financial-eco` vuelve al

## 3. Desacoplamiento aplicado en PR #4

1. `odata-write.routes.ts`: key `endpointServices` por `getBaseModel().getModelName()`.
2. `batch.middleware.ts`: `resolveTarget` usa el último segmento tras `/` como entitySet
   para el `QueryParser`; `Location`/errores usan el nombre de modelo.
3. `odata-metadata.ts`: EntitySet name = nombre del modelo (kebab de `getModelName()`),
   independiente del endpoint/prefijo.

Tras el fix, F0.1 se re-ejecuta: aplicar prefijo + migrar los 67 asserts de test →
`pnpm test` debe quedar en 143+ passed y el demo UI5 debe bootstrappear contra
`/odata/demo/product-odata` sin cambios en el nombre del EntitySet.
