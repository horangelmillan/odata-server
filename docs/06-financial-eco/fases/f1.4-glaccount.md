# F1.4 — Dominio `glaccount` (Cuenta mayor)

> **Sub-fase:** F1.4 · **Esfuerzo:** Bajo · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F1.1.

## 0. Objetivo
Crear `core/glaccount/` — cuenta mayor (G/L Account) tipo S/4HANA, usada por las líneas de factura.

## 1. Modelo `core/glaccount/model/glaccount.odata.model.ts`
```typescript
@Table({ tableName: "glaccounts" })
export class GlAccountOData extends Model<GlAccountOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) glAccount!: string; // "0000123400"
    @Column({ dataType: DataTypes.STRING }) description!: string;                   // "Ventas de mercancías"
    @Column({ dataType: DataTypes.STRING }) accountType!: string;                  // "P" (pérdidas/ganancias) | "B" (balance)
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;

    @HasMany(() => InvoiceItemOData, { relation: [{ foreignKey: "glAccount", sourceKey: "glAccount" }] })
    items!: InvoiceItemOData[];
}
```
(El `@HasMany` a `InvoiceItemOData` se completa en F1.7.)

## 2. Resto del módulo
- `interface/glaccount.interface.ts` → `IGlAccount`.
- `dto/glaccount.dto.ts` → `GlAccountCreateDTO` / `GlAccountUpdateDTO`.
- `controller/glaccount.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/glaccount.service.ts` → singleton.
- `main.ts` → barrel.

## 3. Registro
- `datasource.ts`: añadir `GlAccountOData`.
- `odata.service.ts`: añadir `new GlAccountODataController()`.
