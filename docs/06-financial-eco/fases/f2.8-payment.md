# F2.8 — Seed `payment`

> **Sub-fase:** F2.8 · **Esfuerzo:** Medio · **Depende de:** F2.5, F2.6.

## 0. Objetivo
`scripts/seed/data/payments.json` con ~30 pagos (`P00001`…). Cada pago "clearea" una factura
(cliente → `invoice`, proveedor → `supplierinvoice`) vía `clearedInvoice` + `partnerType`.

Reglas:
- ~60% de facturas cliente tienen pago (`partnerType: "C"`, `clearedInvoice` apunta a `I0000x`,
  `amount` ≈ `grossAmount`).
- Algunos pagos parciales (amount < gross) para realismo.
- Fechas de pago posteriores a la factura y ≤ hoy.

```json
[
  { "paymentId": "P00001", "company": "1000", "partnerId": "C00001", "partnerType": "C",
    "date": "2026-03-10", "amount": 1210.00, "method": "TRANSFER", "clearedInvoice": "I00001",
    "createdAt": "2026-03-10T00:00:00.000Z", "updatedAt": "2026-03-10T00:00:00.000Z" }
]
```

## 1. Criterios de aceptación
- [ ] ~30 pagos con IDs fijos; clearing coherente con facturas PAGADAS.
- [ ] `pnpm seed` reproduce idéntico.
- [ ] F2 global cumplida: `pnpm db:reset` recrea el ecosistema idéntico.
