# F1.1 — Dominio `company` (Sociedad)

> **Sub-fase:** F1.1 · **Esfuerzo:** Bajo · **Patrón:** `docs/02-patrones/05-odata-module-pattern.md`
> **Depende de:** F0.

## 0. Objetivo
Crear `core/company/` — la raíz del ecosistema (sociedad/company code tipo S/4HANA).

## 1. Modelo `core/company/model/company.odata.model.ts`
```typescript
@Table({ tableName: "companies" })
export class CompanyOData extends Model<CompanyOData> {
    @Column({ dataType: DataTypes.STRING, isPrimaryKey: true }) code!: string; // "1000"
    @Column({ dataType: DataTypes.STRING }) name!: string;                     // "Empresa Demo S.L."
    @Column({ dataType: DataTypes.STRING }) currency!: string;                 // "EUR"
    @Column({ dataType: DataTypes.STRING }) country!: string;                 // "ES"
    @Column({ dataType: DataTypes.DATE }) createdAt!: Date;
    @Column({ dataType: DataTypes.DATE }) updatedAt!: Date;
}
```
## 2. Resto del módulo
- `interface/company.interface.ts` → `ICompany`.
- `dto/company.dto.ts` → `CompanyCreateDTO` / `CompanyUpdateDTO` (class-validator).
- `controller/company.odata.controller.ts` → `ODataControler` (`allowedMethod: ["get"]`).
- `service/company.service.ts` → singleton orquestador (lectura vía controller).
- `main.ts` → barrel (modelo + controlador).

## 3. Registro
- `datasource.ts`: añadir `CompanyOData` a `models: [...]`.
- `odata.service.ts`: añadir `new CompanyODataController()` a `odataControllers`.

## 4. Criterios de aceptación
- [ ] `core/company/` completo; `GET /odata/company` responde (vacío hasta el seed en F2).
- [ ] `pnpm test` en verde.
